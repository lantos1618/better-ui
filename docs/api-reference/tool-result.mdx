## Behavior

ToolResult handles several concerns:

1. **View resolution**: Looks up `tools[toolName]` and renders its `.View` component
2. **State store seeding**: Seeds the store with `entityId`, `toolInput`, and `seqNo`
3. **Grouping (anchor/followup)**: When `groupKey` is set, earlier calls with the same `entityId` become "anchors". Newer calls (followups) update the anchor's output data in-place and render nothing â€” the anchor always shows the latest data
4. **HITL flow**: Shows an Approve/Reject card when `shouldConfirm(input)` returns true and args are fully available
5. **Auto-approve**: When `shouldConfirm(input)` returns false but `requiresConfirmation` is true, automatically approves and executes
6. **Error state**: Shows an error card with a Retry button when a tool execution fails
7. **Loading state**: Shows the tool name with a loading indicator during execution
