## Behavior

ToolResult handles several concerns:

1. **View resolution**: Looks up `tools[toolName]` and renders its `.View`
2. **State store seeding**: Seeds the store with `entityId`, `toolInput`, `seqNo`
3. **Grouping collapse**: Uses `useIsCollapsed` â€” renders a chip if superseded by a newer call
4. **HITL flow**: Shows Approve/Reject when `shouldConfirm(input)` returns true
5. **Auto-approve**: When `shouldConfirm(input)` returns false but `requiresConfirmation` is true, auto-approves
6. **Loading state**: Shows tool name with loading indicator during execution
