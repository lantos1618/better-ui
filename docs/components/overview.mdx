Better UI ships view components you can use inside your tool `.view()` functions. These handle common patterns like questions, forms, data tables, and more.

## Usage Pattern

All view components follow the same pattern â€” use them inside your tool's `.view()`:

```tsx
import { QuestionView } from '@lantos1618/better-ui/components';

myTool.view((data, state) => (
  <QuestionView
    question={data.question}
    options={data.options}
    selected={data.selected}
    onSelect={(value) => state?.onAction?.({ ...data, selected: value })}
    loading={state?.loading}
  />
));
```

## Chat Components

Better UI also provides pre-built chat layout components:

| Component | Description |
|-----------|-------------|
| [`Chat`](/api-reference/chat-provider) | Full chat UI (Thread + Composer + Panel) |
| [`ChatProvider`](/api-reference/chat-provider) | Context provider for chat state |
| [`Thread`](/api-reference/thread) | Message list with auto-scroll |
| [`Composer`](/api-reference/composer) | Message input with send button |
| [`Panel`](/api-reference/panel) | Side panel for tool outputs |
| [`Message`](/api-reference/message) | Single message renderer |
| [`ToolResult`](/api-reference/tool-result) | Tool output renderer (auto-resolves views) |
| [`Markdown`](/api-reference/thread) | Markdown renderer for assistant messages |
